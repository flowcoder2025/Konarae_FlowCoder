# Unit eco  
  
Role: **유닛이코노믹스/IR 수치 설계자(프라이머 심사역 관점)**  
  
좋아. **A안(건당 결제)** 기준으로, **작성·검증 모두 Standard**만 쓰는 형태로 IR에 그대로 넣을 수 있게 **모든 수치(가격–원가–마진–연간 ARPA/LTV–CAC 회수–AARRR)**를 “계산식까지” 정리해줄게.  
(환율·GPT-5.2 단가는 최신 공식 페이지 기준으로 반영)    
  
⸻  
  
**Step 1) 3인 패널 합의(각 1스텝)**  
	•	**Expert A(심사역)**: IR에는 “한 장에 읽히는 기준 시나리오(베이스)” + “민감도(10/15/20건)”만 있으면 충분.  
	•	**Expert B(수익모델)**: 원가비율* 20%* 요구가 있었으니, **결제수수료 포함 COGS가 20% 이하**인지 **건당**으로 검증 필요.  
	•	**Expert C(원가/LLM)**: 5분급 작성은 변동이 커서 **안전계수(리비전/재시도) 3배**를 “효과 원가”로 잡아야 IR이 안전.  
  
→ 세 의견을 합쳐 **(1) 가격/크레딧 고정 → (2) GPT-5.2 원가 산식 → (3) 건당/연간 유닛이코노믹스 → (4) AARRR 수치** 순서로 확정.  
  
⸻  
  
**Step 2) 입력값(IR에 박을 고정 가정)**  
  
**2-1. 가격(=A안, 건당 결제) + 크레딧 표기**  
	•	**1C = 200원**  
	•	**검증 Standard = 15C = 3,000원**  
	•	**작성 Standard = 60C = 12,000원**  
	•	(참고) Pro: 검증 30C, 작성 120C (IR에서는 “업셀 옵션”으로만 표기)  
  
**2-2. 결제수수료**  
	•	**f = 3.5%** (국내 PG 평균 수준 가정; 네 PG가 다르면 f만 바꾸면 전체 자동 수정)  
  
**2-3. GPT-5.2 API 단가(텍스트)**  
	•	Input **$1.75 / 1M tokens**  
	•	Cached input **$0.175 / 1M tokens**  
	•	Output **$14.00 / 1M tokens**    
또한 “추론(Reasoning) 토큰”은 **Output으로 과금되는 구조**라서, 장문/리서치형은 Output 관리가 핵심이야.    
  
**2-4. 환율(원가 계산용)**  
	•	1 USD ≈ **₩1,442**    
  
**2-5. 토큰 가정(시간 가정 → IR용 표준 토큰으로 치환)**  
  
너가 준 “5분 / 1분”을 IR에 쓰기 좋은 형태로 표준화:  
	•	**검증 Standard(1분급)**: 입력 25k / 출력 4k  
	•	**작성 Standard(5분급)**: 입력 80k / 출력 20k  
  
실제 운영에선 리비전/재시도가 생기므로 **안전계수 m=3.0**을 곱해 “효과 원가”로 잡음(투자자 방어용).  
  
⸻  
  
**Step 3) GPT-5.2 기준 “건당 AI 원가” 계산(산식 포함)**  
  
**3-1. 기본 산식**  
	•	입력비용(USD) = (Input tokens / 1,000,000) × 1.75  
	•	출력비용(USD) = (Output tokens / 1,000,000) × 14.00  
	•	총비용(KRW) = (입력비용 + 출력비용) × 1,442  
	•	**효과 AI원가 = 총비용 × m(=3.0)**  
  
**3-2. 검증 Standard(25k in / 4k out)**  
	•	입력: 25,000 × 1.75 / 1M = **$0.04375**  
	•	출력: 4,000 × 14 / 1M = **$0.05600**  
	•	합계: **$0.09975** → KRW: 0.09975×1,442 ≈ **₩143.84**  
	•	효과 AI원가: 143.84×3 ≈ **₩431.52**    
  
**3-3. 작성 Standard(80k in / 20k out)**  
	•	입력: 80,000 × 1.75 / 1M = **$0.14000**  
	•	출력: 20,000 × 14 / 1M = **$0.28000**  
	•	합계: **$0.42000** → KRW: 0.42×1,442 ≈ **₩605.64**  
	•	효과 AI원가: 605.64×3 ≈ **₩1,816.92**    
  
⸻  
  
**Step 4) 건당 유닛이코노믹스(가격–수수료–원가–마진)**  
  
**4-1. 검증 Standard (3,000원)**  
	•	매출 P = **₩3,000**  
	•	PG 수수료 = 3.5%×3,000 = **₩105**  
	•	AI 효과 원가 = **₩431.52**  
	•	**COGS(수수료+AI) = ₩536.52 → 원가비율 17.9%**  
	•	**건당 총이익 = 3,000 – 536.52 = ₩2,463.48 (GM 82.1%)**  
  
**4-2. 작성 Standard (12,000원)**  
	•	매출 P = **₩12,000**  
	•	PG 수수료 = 3.5%×12,000 = **₩420**  
	•	AI 효과 원가 = **₩1,816.92**  
	•	**COGS = ₩2,236.92 → 원가비율 18.6%**  
	•	**건당 총이익 = 12,000 – 2,236.92 = ₩9,763.08 (GM 81.4%)**  
  
✅ 결론: 지금 네가 제시한 가격(검증 3천/작성 1.2만)은 **“결제수수료 포함 원가비율 20% 이하”**를 **충분히 만족**.  
  
⸻  
  
**Step 5) 연간 유닛이코노믹스(1유저=1기업) — IR용 3개 시나리오**  
  
베이스** 시나리오 정의(현실적인 가정)**  
	•	연간 지원서 제출 **N건** (10 / 15 / 20)  
	•	**검증은 매 건 수행(=N회)**  
	•	**작성은 30%에서만 수행**(초안 재사용/소폭 수정 고려)  
	•	작성 횟수 = round(0.3×N)  
  
“작성도 매번 한다”는 공격 시나리오는 아래에 민감도로 같이 제공.  
  
⸻  
  
**5-1. 베이스(작성 30%) 결과표**  
  
**연간 제출 N**	**작성 횟수**	**연매출(원)**	**AI원가(원)**	**PG수수료(원)**	**COGS합(원)**	**연총이익(원)**	**GM**  
10	3	66,000	9,766	2,310	12,076	53,924	81.7%  
15	4	93,000	13,740	3,255	16,995	76,005	81.7%

IR에 “ARPA(연매출/기업)”는 위 연매출 그대로 쓰면 됨.  
  
⸻  
  
**5-2. 공격(작성 100%: 매 건 작성+검증) 민감도**  
  
**연간 제출 N**	**작성 횟수**	**연매출(원)**	**연총이익(원)**	**GM**

**Step 6) CAC/LTV(심사역이 꼭 보는 3개 숫자)**  
  
너 이전 자료에서 **CAC 55,000원**을 이미 썼던 흐름이 있어서(그 숫자를 그대로 활용하는 형태로 계산해줄게).  
IR에는 보통 **Payback + LTV/CAC**만 깔끔히 넣으면 충분.  
  
**6-1. Payback(개월) = 12×CAC / 연총이익**  
	•	베이스(작성 30%, N=15): 연총이익 76,005 → **Payback 8.7개월**  
	•	베이스(N=10): **12.2개월**  
	•	베이스(N=20): **6.1개월**  
  
**6-2. LTV 가정(연 유지율 기반)**  
  
IR에서 깔끔한 방식:  
	•	연 유지율(=Retention) **70%** 가정  
	•	기대 이용기간(년) ≈ 1 / (1-0.7) = **3.33년**  
	•	**LTV(총이익 기준) = 연총이익 × 3.33**  
  
예) 베이스(N=15): 76,005×3.33 ≈ **253,095원**  
→ **LTV/CAC = 253,095 / 55,000 ≈ 4.6배**  
  
⸻  
  
**Step 7) AARRR 프레임워크(수치까지 IR에 넣을 수 있게)**  
  
아래는 “지표 정의 + 계산에 바로 쓰는 형태”로 정리했어.  
(초기엔 데이터가 없으니 **가정값**으로 시작하되, *단위경제(ARPA/GM)*는 위에서 이미 고정 수치로 “증명” 가능한 게 강점이야.)  
  
**A — Acquisition (유입)**  
	•	지표: 방문자, 회원가입, 기업프로필 생성  
	•	추천 IR 표기:  
	•	방문자→가입 전환율, 가입당 비용(CPA), 기업프로필 생성율  
  
**A — Activation (활성)**  
	•	**활성 이벤트 정의(추천)**: “첫 프로젝트 생성 + 첫 검증 결제(3,000원)”  
	•	지표:  
	•	Activation Rate = 활성 기업 / 가입 기업  
	•	Time-to-Value = 가입 후 첫 검증까지 소요시간  
  
**R — Retention (유지)**  
	•	지표(둘 중 하나만 고정해서 쓰기):  
	1.	**30일 재구매율**: 첫 결제 후 30일 내 추가 결제 비율  
	2.	**연 유지율**: 다음 해에도 최소 1건 이상 결제한 비율(=LTV 가정과 직결)  
  
**R — Referral (추천)**  
	•	지표:  
	•	초대당 가입 전환율  
	•	K-factor(간단형): K ≈ (유료고객 중 추천비율)×(추천당 평균 초대 수)×(초대→유료 전환율)  
  
**R — Revenue (수익)**  
	•	**ARPA(연간 기업당 매출)**: 위 시나리오 연매출  
	•	**Gross Margin**: 81.5~81.7%(위 계산값)  
	•	**Payback**: 6~12개월대(베이스 범위)  
  
⸻  
  
**IR에 바로 붙여넣는 “숫자 블록”(추천 베이스 1개)**  
  
너 IR에서 가장 설득력 좋은 베이스는 보통 이거야:  
	•	“기존 5~~10건 → 제품 사용 후 10~~20건” 이야기와도 맞고,  
	•	너무 과격하지도 않음.  
  
**[Unit Economics — Base Case]**  
	•	Pricing: Validation Standard **₩3,000 (15C)** / Writing Standard **₩12,000 (60C)**  
	•	GPT-5.2 COGS(결제수수료 포함): Validation **17.9%**, Writing **18.6%**    
	•	Annual per company(1 user=1 company), N=15 submissions/year:  
	•	Revenue **₩93,000/yr**  
	•	Gross Profit **₩76,005/yr**  
	•	Gross Margin **81.7%**  
	•	Payback (CAC ₩55,000 가정): **8.7개월**  
	•	LTV(연 유지율 70% 가정): **₩253,095**, LTV/CAC **4.6x**

**(21 / 33) Unit Economics**  
  
**짧은 카피(헤더 아래 한 줄 또는 박스 캡션용)**  
“**건당 마진이 남는 구조(원가율 20%)로, 제출 횟수가 늘수록 수익이 커집니다.**”    
  
**수치 6개(상단 3개 하이라이트 박스 + 하단 카드/주석 분산 추천)**  
	1.	**Gross Margin = 80%** *(*원가율* 20% *가정* *반영*)*  
	2.	**평균 Credit/건 = 75C** *(*작성* 60C + *검증* 15C)*  
	3.	**건당 매출 = 15,000원** *(75C × 200*원*)*  
	4.	**건당 원가 = 3,000원** *(*매출의* 20%)*  
	5.	**건당 공헌이익 = 12,000원** *(15,000 - 3,000)*  
	6.	**연간 매출/기업 = 225,000원** *(*연* 15*건* *제출* *가정*: 10~20*건의* *중간값*)*  
  
레이아웃 팁: v5 상단 3박스(현재 68%/65C/6,500원)를 **80% / 75C / 15,000원**으로 교체하고, 하단 “가치 정렬 구조” 카드의 작은 글씨로 **원가 3,000원(20%) / 공헌이익 12,000원 / 연매출 225,000원** 3개를 넣으면 “6개 숫자”가 깔끔하게 소화됩니다.
